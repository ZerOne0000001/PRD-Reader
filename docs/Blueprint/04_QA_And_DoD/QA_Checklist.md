# ✅ 交互交付前的严格自测验证清单 (QA Checklist)

在修改任何具有交互逻辑的页面（如点击目录跳转、弹窗唤起、状态切换等）后，**严禁直接交接给用户验收**。AI 必须自行通过本清单进行代码逻辑走查，确保 100% 连通无死角后，方可请用户确认。

## 1. 路由与 DOM 映射检查
*   [ ] **URL 参数与状态同步**：修改后的组件是否正确读取了 URL 中的参数（如 `?repo=xxx&file=xxx`）并同步到了 Zustand Store？反之，Store 的变化是否正确反映到了 URL 上（支持复制分享）？
*   [ ] **锚点跳转匹配**：TOC 目录生成的 `href` `#id` 是否与 Markdown 渲染后生成的 H1/H2 标签的 `id` **完全匹配**？
*   [ ] **虚拟静态路由测试**：新增加的 HTML 或图片预览，是否正确使用了 `/api/repo/raw/:projectId/*` 路由？前端直接构造的 `src` 是否做了正确的 URL Encode？

## 2. 状态机闭环检查 (Zustand)
*   [ ] **加载状态闭环**：所有异步操作是否都正确处理了 `loading` 状态的开启与关闭？
*   [ ] **异常状态恢复**：如果接口报错，错误信息是否正确写入了 `error` 状态并在 UI 上展示？
*   [ ] **脏数据清理**：从 A 仓库切换到 B 仓库时，前一个仓库的 `currentFile` 内容是否被正确清空，避免出现“闪烁旧数据”的 Bug？

## 3. 视觉与交互微细节
*   [ ] **滚动条规范**：是否严格遵守了“全站强制隐藏滚动条（`::-webkit-scrollbar { display: none; }`）但保留滚动能力”的全局规范？
*   [ ] **剪贴板反馈**：所有“复制”操作，点击后是否提供了“变为绿色打勾图标（停留 2 秒）后恢复”的明确视觉反馈？
*   [ ] **抽屉交互**：左侧文件树的“滑出/收起/固定”逻辑是否存在冲突？

## 4. 后端安全与异常边界
*   [ ] **白名单越权测试**：后端路由是否严格校验了请求的 `projectId` 是否在 `config.json` 的对应平台白名单内？
*   [ ] **大文件崩溃测试**：对于非 `.md` 的大型文件，是否依然错误地将其读取到内存并通过 JSON 返回？（必须通过 raw 流式/直接静态托管处理）。
