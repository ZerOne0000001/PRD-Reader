# 🎨 UI/UX 增量草案：阅读区功能 (F02_Reader)

> 本文件是基于迭代 2604_2（分享文档链接）提取的产品沙箱草案，供开发实现时参考。

## 1. 左侧抽屉式文件树 (Drawer Tree) - 交互改造
*   **分享交互区 (Popover Menu)**：
    *   **图标触发机制**：原有的“复制地址”交互逻辑修改。现在鼠标悬停在左侧树任意节点（仓库、文件夹、文件）时，出现统一的“分享 Icon”（`lucide-copy`）。
    *   **气泡菜单 (Popover)**：
        *   点击分享 Icon 后，弹出一个小型气泡选项菜单。
        *   菜单项包含：【复制 Git 原地址】（仅针对 GitHub/GitLab 支持平台）和【复制系统分享链接】。
        *   气泡菜单必须带有关闭遮罩（点击页面其他区域或选中任一选项时自动关闭）。
    *   **视觉反馈**：点击复制后，在界面上弹出轻量级的 Toast 提示（如“已复制系统分享链接”）。
*   **下钻模式下的导航栏 (Drill-down Navigation)**：
    *   当系统处于分享链接的下钻状态时，左侧树顶部需提供一个包含“返回上级”按钮和当前下钻路径文本的区域。
    *   按钮视觉应明显，供用户点击恢复完整的全局仓库列表。

## 2. 右侧主阅读区 (Main Content Area) - 新增状态
*   **404 缺省状态页 (Empty State / In-place 404)**：
    *   当通过分享链接访问的内容存在异常时，在主阅读区居中渲染缺省页。
    *   **状态 1：仓库无权限/未配置**
        *   视觉设计：粉色底框 + 锁头 Icon (`lucide-lock`)。
        *   文案：主标题“仓库无权限或未配置”；副标题说明无权限的原因及操作指引。
        *   操作按钮：“返回全局视图”（点击后清除 URL 参数，恢复默认状态）。
    *   **状态 2：文件/路径不存在**
        *   视觉设计：灰色底框 + 未知文件 Icon (`lucide-file-question`)。
        *   文案：主标题“文件不存在或已被删除”；副标题说明可能被重命名或移除。
        *   操作按钮：“浏览该仓库其他文件”（点击后重置树结构并清除 URL 的 `file` 参数）。
*   **Loading 状态**：
    *   接收者通过链接首次进入时，在数据加载期间，需展示居中的 Loading 骨架屏或 Spinner（文案如“正在定位分享文档...”）。
