# ⚖️ 业务规则增量草案：阅读区功能 (F02_Reader)

> 本文件是基于迭代 2604_2（分享文档链接）提取的产品沙箱规则增量，供开发实现时参考。

## 1. 树状数据处理规则 (增量修改)
*   **下钻模式 (Drill-down View)**：
    *   **前置条件**：页面初始化时，若 URL 中带有特定的 `file` 和 `repo` 参数。
    *   **下钻规则**：
        *   若分享目标为**文件夹**，系统以该文件夹为唯一根节点进行渲染（隐藏全局仓库列表）。
        *   若分享目标为**具体文件**，系统以该文件所在的**父文件夹**为唯一根节点进行渲染，并在加载完成后高亮选中该文件节点。
    *   **状态清除**：用户点击“返回上级”时，清除下钻状态（重置 `drillDownRoot`），并恢复渲染全局仓库列表。

## 2. 状态机与 URL 同步规则 (新增)
*   **单向同步优先 (State to URL)**：
    *   用户在左侧文件树点击任意合法节点导致当前阅读状态（`currentFile` 或选中节点）发生变化时，状态机必须**静默更新**浏览器的 Query Params（即 `?repo=xxx&file=yyy`）。
    *   此过程仅修改 History API，**绝对不允许**触发页面的整体刷新或重载。
    *   若用户点击的是仓库根节点（非文件/文件夹），则 URL 中仅保留 `repo` 参数，移除 `file` 参数。
*   **初始化挂载拦截 (URL to State)**：
    *   系统仅在首次加载（Mount）时，读取一次 URL 的 Query Params。
    *   如果有参数，状态机优先根据参数发起异步请求（拉取指定仓库、定位文件路径、执行下钻并渲染内容）。

## 3. 渲染分支规则 (增量修改)
*   **原位异常拦截 (In-place 404)**：
    *   如果初始化根据 URL 请求数据失败，**不触发全局路由跳转**。
    *   **权限校验失败（403 或 repo 不在白名单）**：在右侧主阅读区内直接渲染“仓库无权限”状态页。
    *   **文件查找失败（404 或路径在内存树中不存在）**：在右侧主阅读区内直接渲染“文件不存在”状态页。
    *   在判断 404 或发起请求期间，必须先在主阅读区展示全局 Loading 骨架屏，防止 UI 闪烁。
