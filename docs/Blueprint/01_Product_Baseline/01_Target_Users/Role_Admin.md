# 🛠️ 角色画像：系统管理员 (System Admin)

## 1. 角色定位
负责 PRD-Reader 系统的底层配置与运维。通常由项目经理、技术架构师或运维人员兼任。

## 2. 核心诉求
*   **安全可控**：能够安全地配置底层 GitLab 实例的访问 Token，不向外部暴露。
*   **权限隔离**：能够精准控制哪些 GitLab 仓库可以暴露给前台（白名单机制）。
*   **管理便捷**：配置界面必须简单直接，只需一次配置，即可长期稳定运行。

## 3. 权限边界
*   **唯一拥有**后台管理界面（Admin Dashboard）的访问权限。
*   能够修改全局实例地址、全局访问令牌。
*   能够增删查改前台可见的仓库白名单列表。
*   默认拥有 `Role_Reader` 的所有权限。

## 4. 关键交互触点
*   通过特殊的隐藏入口（如底部版本号点击）或特定路由进入后台。
*   需要输入硬编码的管理员密码（目前为 `Aa@000000`）进行身份校验。
