# 👁️ 角色画像：只读用户 (Reader)

## 1. 角色定位
PRD-Reader 系统的核心服务对象。通常是产品经理、测试人员、非核心开发人员或项目干系人。

## 2. 核心诉求
*   **零摩擦访问**：不需要拥有 GitLab 账号，也不需要懂 Git 的复杂操作，甚至不需要登录系统。打开链接就能直接看到想看的文档或原型。
*   **沉浸式体验**：只关心文档的内容、排版和原型交互。不需要看到任何与代码库相关的繁杂信息（如 commit 记录、CI/CD 状态等）。
*   **高效定位**：通过左侧直观的树状目录快速找到目标文件。
*   **信息共享**：能够将当前正在阅读的文档或原型，通过 URL 一键分享给其他成员，对方打开即看，状态保持一致。

## 3. 权限边界
*   **绝对只读**：只能浏览和阅读由 `Role_Admin` 配置的白名单仓库中的文件。
*   **无权修改**：无法对文档内容进行任何修改、提交或删除操作。
*   **无权配置**：无法访问后台管理界面，无法查看或修改系统底层的 GitLab 实例地址、全局访问令牌或仓库白名单。

## 4. 关键交互触点
*   系统首页（文件树浏览）。
*   Markdown 沉浸式阅读视图（带侧边 TOC）。
*   HTML 原型高保真隔离预览视图。
*   图片文件的直接渲染展示。
