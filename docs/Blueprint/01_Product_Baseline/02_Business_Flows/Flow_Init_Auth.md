# 🔐 流程：初始化与鉴权闭环 (Flow_Init_Auth)

## 1. 流程描述
本流程定义了系统管理员如何配置 PRD-Reader 的底层 GitLab 访问能力，确保系统的基建畅通。

## 2. 步骤拆解
1.  **系统管理员进入**：首次或后续进入系统，通过底部特定入口（如点击版本号）进入后台登录页。
2.  **身份校验**：输入预设的管理员密码（如 `Aa@000000`）。
3.  **配置实例基建**：配置底层 GitLab 的实例地址以及具有 `read_api` 权限的全局访问令牌 (Global Token)。
4.  **配置仓库白名单**：添加允许在前端展示的 GitLab 仓库列表（通过 Project ID 或 Path）。

## 3. 业务约束
*   **统一代理**：前台所有的 GitLab API 请求（包括拉取目录树、文件内容）必须通过本系统的 Node.js 后端代理进行，以附带全局 Token。前端绝对不能直接请求 GitLab API，以防 Token 泄漏。
*   **白名单隔离**：后端在处理获取文件内容的请求时，必须校验请求的 `projectId` 是否在白名单中。不在白名单的仓库绝对不可访问。
