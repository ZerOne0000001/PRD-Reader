# ⚖️ 业务规则：阅读区功能 (F02_Reader)

## 1. 树状数据处理规则
*   **聚合与转换**：后端返回扁平化的 Git 树结构，前端需根据 `path` 字段自行重建层级嵌套的树状结构。
*   **单仓库限制**：每次调用 `fetchTree` 时，必须附加当前所在的仓库参数 `?repo=xxx`，确保内存中只加载当前目标仓库的结构。
*   **下钻模式 (Drill-down View)**：
    *   当分享链接带有 `file` 参数，如果目标是**文件夹**，系统以该文件夹为唯一根节点进行渲染；如果是**具体文件**，系统以该文件所在的**父文件夹**为唯一根节点渲染。
    *   提供“返回上级”按钮退出下钻模式。
*   **排序规则**：同一目录下，文件夹 (Tree) 排在最前，文件 (Blob) 排在后面。同类型内按名称字母顺序升序排列。

## 2. 状态机与 URL 同步规则
*   **静默同步优先**：用户点击左侧文件树导致阅读状态变化时，必须通过 History API (`replaceState`) 静默更新浏览器 URL (`?repo=xxx&file=yyy`)，不触发整体刷新。
*   **初始化拦截**：系统仅在首次加载或 `repo` 变更时，读取 URL 参数并发起数据请求，默认尝试拉取并渲染根目录的 `README.md`。

## 3. 文件访问与安全
*   **私有仓库保护**：仅支持访问配置白名单内的仓库（依赖后端的拦截与 GitHub/GitLab 提供的 Token 权限）。
*   **沙箱隔离**：渲染 `.html` 时必须使用 `iframe`，且通过 `sandbox` 属性防止潜在的 XSS 攻击影响主应用。
*   **原位异常拦截 (In-place 404)**：如果无权限或文件不存在，不触发路由跳转，在主阅读区直接接管异常渲染。
