# 🎨 UI/UX：后台配置 (F03_Admin)

## 1. 访问入口
*   **入口位置**：首页左侧边栏底部的“设置”齿轮图标。
*   **鉴权弹窗交互**：
    *   点击后弹出全屏毛玻璃遮罩层及密码输入框。
    *   输入框带有锁图标，密码掩码显示。
    *   **错误反馈**：输入错误点击确认（或回车）时，输入框边框变红，下方展示红色错误文案，且弹窗本体触发轻微的“震动（Pulse）”动画反馈。

## 2. 平台切换区 (Tab)
*   顶部居中显示 GitLab 和 GitHub 两个平台的切换 Tab。
*   切换平台时，下方配置区域（Token 与白名单）自动更新为对应平台的数据。
*   当前平台的选择会被持久化保存。

## 3. 全局配置区
*   **输入框**：
    *   实例地址：带平台特性的占位符（如 `https://gitlab.com` 或 `https://github.com`）。
    *   访问令牌：密码掩码形式输入，右侧提供“小眼睛”图标切换明文/密文显示。
*   **状态指示灯**：输入区旁带有系统连接状态（绿色呼吸灯圆点 + “已连接”字样，或红色圆点 + “异常”）。
*   **交互按钮**：
    *   测试连接：点击后实时反馈结果。
    *   保存配置：保存按钮在鼠标悬浮时带有向上微移的物理投影效果。

## 4. 仓库白名单管理区
*   **列表展示**：左侧展示仓库名称缩写（生成的 Logo）及完整路径，右侧展示连接状态（绿色“正常”或红色“异常”）。
*   **移除交互**：
    *   仅当鼠标悬停在对应仓库卡片时，右侧显现红色的垃圾桶图标。
    *   点击垃圾桶直接移除，**不弹出**二次确认对话框（追求极致高效）。
