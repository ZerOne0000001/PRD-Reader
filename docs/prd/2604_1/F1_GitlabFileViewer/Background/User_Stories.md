# 用户故事 (User Stories)

## 史诗 (Epic)
构建一个基于 GitLab 只读账号的 Web 文件查看平台，核心解决产品和技术人员在查阅跨仓库 Markdown 文档、Mermaid 图表以及单文件 HTML 时的阅读体验痛点，并提供偏图形化的交互。

## 故事列表

### 1. 配置可见仓库白名单
**作为** 系统管理员
**我想要** 能够配置该平台可以访问的 GitLab 仓库列表
**以便于** 我可以控制前端用户只能查看到指定的文档仓库，避免暴露核心业务代码，减少信息噪音。

**验收标准 (Acceptance Criteria):**
* [ ] 提供一个后台配置页面，允许管理员输入需要开放给该平台的 GitLab 仓库的 Path 或 ID。
* [ ] 系统使用预先配置的全局只读账号进行 GitLab API 认证。
* [ ] 如果配置的仓库不存在或账号无权限，后台应提示配置失败。
* [ ] 前台界面的文件树仅展示该白名单内的仓库。

### 2. 跨项目全文搜索
**作为** 产品经理或技术人员
**我想要** 在一个全局搜索框中输入关键词，对所有已配置白名单的仓库进行全文搜索
**以便于** 我能快速找到散落在各个业务线仓库中的需求文档和原型。

**验收标准:**
* [ ] 首页或顶部导航栏提供一个显眼的搜索框。
* [ ] 输入关键词后，能够跨越所有配置的白名单仓库，对文件内容（如 Markdown、HTML 内容）和文件名进行搜索。
* [ ] 搜索结果列表需展示匹配的文件名、所属仓库以及内容片段的高亮预览。
* [ ] 点击搜索结果项，可以直接跳转并打开该文件详情页。

### 3. 图形化文件树浏览
**作为** 产品经理或技术人员
**我想要** 通过一个左侧的树状目录结构，直观地浏览白名单仓库内的文件夹和文件
**以便于** 我能像操作本地文件系统一样，通过偏图形化的交互方式找到我需要的文档。

**验收标准:**
* [ ] 界面左侧固定显示文件资源管理器（树状结构）。
* [ ] 第一级节点展示已配置的仓库名称，点击可展开其默认分支的目录结构。
* [ ] 支持多级文件夹的展开与折叠。
* [ ] 根据文件扩展名（`.md`, `.html` 等）展示对应的小图标，增强图形化识别度。

### 4. 沉浸式阅读 Markdown 文档 (含 Mermaid)
**作为** 产品经理或技术人员
**我想要** 在点击 Markdown 文件时，右侧主区域能渲染出排版精美的富文本内容，并且能正常解析 Mermaid 图表
**以便于** 我能舒适地阅读 PRD，清晰地查看其中的业务流程图和架构图，无需下载。

**验收标准:**
* [ ] 点击左侧文件树的 `.md` 文件，右侧区域自动请求该文件的原始内容并渲染为 HTML。
* [ ] 渲染后的页面具有良好的排版样式（标题、列表、表格、代码块等）。
* [ ] 如果 Markdown 中包含 ````mermaid` 代码块，必须能够使用 Mermaid.js 或类似库将其渲染为可视化图表。
* [ ] 渲染区域提供“全屏”或“隐藏左侧栏”的选项，以获得更沉浸的阅读体验。

### 5. 预览单文件 HTML 原型
**作为** 产品经理或技术人员
**我想要** 在点击 HTML 文件时，系统能直接渲染出该网页的视觉效果，而不是展示 HTML 源码
**以便于** 我能直接在浏览器中查看交互演示或静态切图，无需将其 clone 到本地。

**验收标准:**
* [ ] 点击左侧文件树的 `.html` 单文件，右侧区域直接将其作为网页渲染（可以通过 iframe 隔离，或直接解析渲染，确保样式隔离和安全性）。
* [ ] 该 HTML 必须能够正常显示其内联的样式和结构（由于是单文件 HTML，不涉及复杂的相对路径资源加载，但必须保证其独立渲染）。
* [ ] 提供“在新标签页打开”该 HTML 演示的按钮，方便全屏演示。
