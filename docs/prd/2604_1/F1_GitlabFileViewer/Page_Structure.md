# 页面结构 (Page Structure)

## 1. 全局导航与结构概览
*   **适配端侧**: 桌面端 (PC) 为主。
*   **UI 风格**: 清新阅读风 (类似 Notion)，左树右内容布局，注重留白与阅读舒适度。
*   **页面列表**:
    *   `index.html`: 平台主页（包含顶部导航搜索、左侧资源树、右侧阅读区/欢迎页）。
    *   `admin.html` (可选): 后台配置页（用于配置可见仓库白名单和 Token，也可做成简单的 JSON 配置文件驱动）。

## 2. 主页面结构 (`index.html`)

### 2.1 顶部导航栏 (Header)
*   **位置**: 页面最上方，固定高度。
*   **元素**:
    *   **Logo/系统名称**: 左侧居中显示“GitLab File Viewer”或公司 Logo。
    *   **全局搜索框 (核心)**: 居中显示，占据较大宽度，带搜索图标。
        *   **交互**: 聚焦或输入时，下方弹出搜索结果面板（按仓库分组，高亮匹配内容）。
        *   **状态**: 搜索中显示 Loading 动画，无结果显示占位图。
    *   **快捷操作区**: 右侧，如“切换明亮/暗黑主题”、“帮助文档”、“固定/取消固定”等。
    *   **交互逻辑**:
        *   默认状态下顶部导航栏隐藏（向上收起）。
        *   用户将鼠标移动至页面最顶部边缘时，导航栏自动滑出展示。
        *   提供“固定(Pin)”按钮，点击后导航栏固定在页面顶部。

### 2.2 左侧文件资源树 (Sidebar - File Explorer)
*   **位置**: 页面左侧，采用**抽屉式隐藏/展开交互**。
*   **交互逻辑**: 
    *   默认状态下侧边栏隐藏（收起至左侧视窗外）。
    *   用户将鼠标移动至屏幕最左侧边缘时，侧边栏自动滑出展开。
    *   展开后，右上角提供“固定(Pin)”按钮，点击后侧边栏固定在页面左侧，右侧主视图向右挤压适应。
    *   取消固定后恢复默认的隐藏悬浮交互。
    *   **下钻导航 (Drill-down)**: 支持点击文件夹右侧的进入按钮，以该文件夹为根目录重新渲染文件树，顶部提供“返回上级”功能，方便深层目录的专注阅读。
*   **元素**:
    *   **系统标题**: 顶部显示 "PRD Reader" 名称及彩色圆点 Logo。
    *   **仓库列表 (根节点)**: 显示所有配置的白名单仓库名称，附带云朵图标，支持点击展开/收起。
    *   **目录/文件层级 (子节点)**:
        *   **多级文件夹**: 支持嵌套的层级结构（树状缩进），带有文件夹图标。鼠标悬停时右侧出现“复制 GitLab 地址”和“进入目录”操作。
        *   **文件**: 带有类型图标（如 `.md` 蓝图标, `.html` 橙图标等）。鼠标悬停时右侧出现“复制 GitLab 地址”按钮。
    *   **底部固定入口**:
        *   提供指向“后台配置”的入口，固定在侧边栏底部。
        *   点击触发管理员密码验证弹窗。
    *   **交互**:
        *   选中节点高亮显示。
        *   点击文件触发右侧阅读区加载。
        *   滚动条样式极简。

### 2.3 右侧主视图/阅读区 (Main Content Area)
*   **位置**: 占据右侧剩余所有空间。
*   **状态 1: 欢迎/空状态 (Empty State)**
    *   刚进入系统且 URL 无文件参数时显示。
    *   展示一句友好的欢迎语和简单的使用说明（如“从左侧选择仓库开始浏览文档，或使用顶部搜索框”）。
*   **状态 2: 加载中 (Loading State)**
    *   请求文件内容时，展示骨架屏 (Skeleton) 或优雅的 Loading 动画。
*   **状态 3: Markdown 阅读视图 (Markdown View)**
    *   **顶部工具栏**: 面包屑导航（展示当前文件路径 `仓库 / 目录 / xxx.md`），右侧提供“在新标签页打开”、“全屏”等按钮。
    *   **内容区**:
        *   渲染后的富文本内容，采用类似 Notion 的排版样式，全局隐藏滚动条（UI 更整洁）。
        *   **内容居中对齐**，行距适中，字体清晰，合理的标题层级。
        *   `mermaid` 代码块自动渲染为 SVG 图表，支持放大查看。
    *   **右侧目录区 (TOC)**:
        *   在主阅读区**最右侧独立展示**目录树，与正文内容互不干扰。
        *   根据 Markdown 的 H1/H2/H3 标题自动生成。
        *   支持点击平滑滚动定位，并跟随阅读进度自动高亮当前层级。
*   **状态 4: HTML 预览视图 (HTML View)**
    *   **顶部工具栏**: 同 Markdown 视图，增加“刷新 iframe”和“独立窗口打开演示”按钮。
    *   **内容区**:
        *   嵌入一个 100% 宽高的 `iframe`。
        *   将获取到的 Raw HTML 内容直接写入 `iframe`，实现安全隔离和真实的网页预览效果。
*   **状态 5: 错误提示 (Error State)**
    *   如文件不存在、网络超时、格式不支持等，展示相应的错误图标和重试按钮。

## 3. 后台配置页面 (`admin.html`)

### 3.1 全局导航与结构
*   **位置**: 独立的后台管理入口，通常仅对管理员开放。
*   **UI 风格**: 简洁表单式，侧重于数据展示与操作反馈。

### 3.2 核心配置项
*   **GitLab 全局 Token 配置区**:
    *   **输入框**: 用于输入具有只读权限的 GitLab Token。
    *   **状态展示**: 显示当前 Token 的有效性（如“已连接”、“校验失败”）。
    *   **操作按钮**: 保存配置、测试连接。
*   **可见仓库白名单管理区**:
    *   **添加仓库**: 提供输入框（通过仓库 ID 或 URL 路径添加），点击“添加”后调用 API 校验仓库是否存在。
    *   **已配置仓库列表**: 表格或列表展示，包含：仓库名称、仓库路径、添加时间、状态（正常/无权限）。
    *   **操作列**: 支持从白名单中“移除”仓库。

## 4. 交互与状态同步细节
*   **URL 参数驱动**: 页面的核心状态必须由 URL 驱动。例如 `?repoId=123&path=docs/API.md`。
    *   当用户在左侧点击该文件时，URL 自动更新。
    *   当用户直接访问该 URL 时，左侧树必须自动展开到对应节点并高亮，右侧直接加载该文件。这对于团队内的分享至关重要。
