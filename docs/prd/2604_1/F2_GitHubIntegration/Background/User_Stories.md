# 用户故事 (User Stories)

## Epic: GitHub 集成功能

---

### US-1: 平台选择器

**作为** 系统管理员  
**我想要** 在后台配置页面选择代码托管平台（GitLab 或 GitHub）  
**以便于** 根据团队需求切换使用不同的平台

#### 验收标准
- [ ] 后台配置页面顶部显示平台选择器（单选按钮或下拉框）
- [ ] 默认选中当前已配置的平台（首次使用默认 GitLab）
- [ ] 切换平台时，下方配置区域自动更新为对应平台的配置项
- [ ] 切换平台时，白名单区域自动更新为对应平台的仓库列表
- [ ] 平台选择状态持久化保存

#### 优先级
P0 - 核心功能

---

### US-2: GitHub 访问令牌配置

**作为** 系统管理员  
**我想要** 配置 GitHub 的实例地址和 Personal Access Token  
**以便于** 系统能够访问 GitHub 仓库

#### 验收标准
- [ ] 选择 GitHub 平台后，显示 GitHub 配置表单
- [ ] GitHub 实例地址输入框，默认值为 `https://github.com`
- [ ] Token 输入框支持密码模式显示/隐藏
- [ ] 提供"测试连接"按钮，验证 Token 有效性
- [ ] 测试连接成功后显示"已连接"状态标识
- [ ] 测试连接失败后显示错误提示（如权限不足、Token 无效等）
- [ ] 提供"保存配置"按钮，保存后配置持久化
- [ ] Token 权限说明：提示需要 `public_repo` + `repo` 权限

#### 优先级
P0 - 核心功能

---

### US-3: GitHub 仓库白名单管理

**作为** 系统管理员  
**我想要** 添加或删除 GitHub 仓库到白名单  
**以便于** 控制前台用户可见的仓库范围

#### 验收标准
- [ ] 白名单区域根据当前选中的平台展示对应仓库列表
- [ ] 支持通过仓库 ID 或路径（如 `owner/repo`）添加仓库
- [ ] 添加仓库时自动验证仓库是否存在且 Token 有访问权限
- [ ] 添加成功后显示仓库名称、路径、默认分支和状态
- [ ] 支持删除白名单中的仓库（hover 显示删除按钮）
- [ ] 重复添加同一仓库时提示"仓库已存在"
- [ ] 仓库不存在或无权限时显示具体错误信息

#### 优先级
P0 - 核心功能

---

### US-4: GitHub 配置持久化

**作为** 系统管理员  
**我想要** 两个平台的配置独立保存  
**以便于** 切换平台时无需重新配置

#### 验收标准
- [ ] GitLab 和 GitHub 的配置信息独立存储
- [ ] GitLab 和 GitHub 的仓库白名单独立存储
- [ ] 切换平台时自动加载对应平台的配置和白名单
- [ ] 配置保存到服务端配置文件（config.json）
- [ ] 配置变更后立即生效，无需重启服务

#### 优先级
P0 - 核心功能

---

### US-5: GitHub 文件树浏览

**作为** PM 或技术人员  
**我想要** 在前台浏览 GitHub 仓库的文件树  
**以便于** 快速定位需要查看的文档

#### 验收标准
- [ ] 前台首页根据当前配置的平台展示对应仓库列表
- [ ] 点击仓库名称展开文件树
- [ ] 支持无限嵌套目录展开/折叠
- [ ] 文件和文件夹显示对应图标
- [ ] 加载状态显示 loading 动画
- [ ] 加载失败时显示错误提示，引导管理员检查配置

#### 优先级
P0 - 核心功能

---

### US-6: GitHub Markdown 文件渲染

**作为** PM 或技术人员  
**我想要** 在系统中渲染 GitHub 上的 Markdown 文件  
**以便于** 获得沉浸式的文档阅读体验

#### 验收标准
- [ ] 点击 `.md` 文件后右侧渲染 Markdown 内容
- [ ] 支持 Mermaid 图表渲染
- [ ] 自动生成文档目录（TOC）
- [ ] 支持 GitHub Flavored Markdown 语法
- [ ] 代码块语法高亮
- [ ] 渲染失败时显示原始内容和错误提示

#### 优先级
P0 - 核心功能

---

### US-7: GitHub HTML 文件预览

**作为** PM 或技术人员  
**我想要** 在系统中预览 GitHub 上的 HTML 文件  
**以便于** 查看高保真原型效果

#### 验收标准
- [ ] 点击 `.html` 文件后在新标签页或 iframe 中渲染
- [ ] 支持单文件 HTML 渲染
- [ ] 渲染环境与源文件隔离，避免样式污染
- [ ] 加载失败时显示错误提示

#### 优先级
P1 - 重要功能

---

### US-8: GitHub 图片文件查看

**作为** PM 或技术人员  
**我想要** 在系统中查看 GitHub 上的图片文件  
**以便于** 查看设计稿或流程图

#### 验收标准
- [ ] 支持 PNG、JPG、SVG、WEBP 等常见图片格式
- [ ] 点击图片文件后直接渲染展示
- [ ] 图片支持缩放和拖拽
- [ ] 加载失败时显示占位图和错误提示

#### 优先级
P1 - 重要功能

---

### US-9: 平台连接异常处理

**作为** 用户  
**我想要** 在平台连接异常时获得清晰的错误提示  
**以便于** 了解问题原因并采取相应措施

#### 验收标准
- [ ] Token 失效时，文件树区域显示"连接失败，请联系管理员检查配置"
- [ ] 仓库不存在或无权限时，显示具体错误信息
- [ ] 网络故障时，显示"网络连接失败，请稍后重试"
- [ ] 错误提示包含可能的解决方案或操作建议
- [ ] 管理员可在后台看到详细的错误日志

#### 优先级
P1 - 重要功能
