# 需求背景 (Requirement Background)

## 1. 业务目标与背景

### 1.1 现状问题
当前 PRD-Reader 系统仅支持 GitLab 平台的文件查看功能。然而，团队在实际工作中可能同时使用 GitHub 托管部分仓库（如开源项目、外部合作项目等）。这导致以下问题：
- 用户需要在 GitLab 和 GitHub 两个平台之间切换，体验割裂
- GitHub 上的文档无法享受 PRD-Reader 提供的沉浸式阅读体验
- 无法统一管理和查阅跨平台的文档资源

### 1.2 业务目标
在现有 GitLab 文件查看系统的基础上，扩展支持 GitHub 平台，实现：
- **平台灵活切换**：管理员可在后台配置选择使用 GitLab 或 GitHub 平台
- **配置独立保留**：两个平台的配置信息（Token、白名单）独立存储，切换时自动加载对应配置
- **功能对等支持**：GitHub 平台支持与 GitLab 相同的核心功能（文件树浏览、Markdown 渲染、HTML 预览、图片查看）

## 4. 功能范围与约束

### 4.1 功能范围
- **平台切换**：支持 GitLab 和 GitHub 两个平台的切换
- **配置管理**：两个平台的配置信息独立存储，切换时自动加载
- **白名单管理**：两个平台的仓库白名单独立管理
- **文件查看**：GitHub 平台支持与 GitLab 相同的文件查看功能（Markdown、HTML、图片）

### 4.2 技术约束
- GitHub 使用 Personal Access Token 进行鉴权
- Token 需具备 `public_repo` 权限（公开仓库）和 `repo` 权限（私有仓库）
- 前端所有 GitHub API 请求必须通过后端代理，禁止前端直接请求
- 后端需校验请求的仓库是否在白名单中

### 4.3 非功能需求
- 平台切换响应时间 < 1 秒
- 配置保存后立即生效，无需重启服务
- 错误提示清晰明确，便于用户理解和处理
