# 用户故事 (User Stories) - F2 文档查询功能

## Story 1: 输入关键词并触发搜索

**作为** 产品经理或技术人员  
**我想要** 在搜索框中输入关键词并点击搜索按钮  
**以便于** 快速检索所有授权仓库中的相关文档

### 验收标准
- [ ] 搜索框支持输入任意文本关键词
- [ ] 搜索框为空时，点击搜索按钮不触发搜索（或提示"请输入搜索关键词"）
- [ ] 点击搜索按钮后，跳转到独立的搜索结果页面
- [ ] 搜索结果页面 URL 包含搜索关键词参数（如 `/search?q=关键词`）
- [ ] 搜索过程中显示加载状态

---

## Story 2: 查看搜索结果列表

**作为** 产品经理或技术人员  
**我想要** 在搜索结果页面看到匹配的文档列表  
**以便于** 快速判断哪些是我需要的文档

### 验收标准
- [ ] 搜索结果页面展示匹配的文档列表
- [ ] 每条结果包含：仓库名、文件路径、文件名
- [ ] 对于 `.md` 文件的内容匹配，展示内容片段预览（关键词前后各 50 字符）
- [ ] 关键词在预览片段中**高亮显示**
- [ ] 对于文件名匹配，展示文件名并高亮匹配部分
- [ ] 页面顶部显示"共找到 X 个结果"统计信息
- [ ] 结果按相关度排序（文件名匹配优先于内容匹配）

---

## Story 3: 分页浏览搜索结果

**作为** 产品经理或技术人员  
**我想要** 对搜索结果进行分页浏览  
**以便于** 在结果较多时不会一次性加载过多内容

### 验收标准
- [ ] 每页默认展示 20 条结果
- [ ] 页面底部显示分页控件（上一页、下一页、页码）
- [ ] 当前页码高亮显示
- [ ] 切换页面时保持搜索关键词不变
- [ ] 分页切换时有加载状态反馈

---

## Story 4: 点击结果打开文档

**作为** 产品经理或技术人员  
**我想要** 点击搜索结果直接打开对应文档  
**以便于** 快速阅读目标文档内容

### 验收标准
- [ ] 点击任意搜索结果，在**新标签页**中打开 Reader 页面
- [ ] Reader 页面自动加载并展示对应的文档内容
- [ ] 左侧文件树自动展开并高亮选中的文件节点
- [ ] 原搜索结果页面保持不变（不关闭）

---

## Story 5: 空结果状态展示

**作为** 产品经理或技术人员  
**我想要** 在没有匹配结果时看到友好的提示  
**以便于** 知道搜索已完成但没有找到相关内容

### 验收标准
- [ ] 当搜索结果为空时，展示空状态插图和提示文案
- [ ] 提示文案："未找到与「关键词」相关的文档，请尝试其他关键词"
- [ ] 提供返回首页或重新搜索的引导

---

## Story 6: 搜索加载状态

**作为** 产品经理或技术人员  
**我想要** 在搜索过程中看到加载状态  
**以便于** 知道系统正在处理我的请求

### 验收标准
- [ ] 点击搜索按钮后，搜索结果页面显示加载动画
- [ ] 加载过程中禁用搜索按钮，防止重复提交
- [ ] 搜索完成后，加载状态消失，展示结果或空状态

---

## Story 7: 搜索结果区分匹配类型

**作为** 产品经理或技术人员  
**我想要** 知道搜索结果是来自文件名匹配还是内容匹配  
**以便于** 更好地判断结果的相关性

### 验收标准
- [ ] 每条结果标注匹配类型标签："文件名匹配" 或 "内容匹配"
- [ ] 文件名匹配的结果排在内容匹配之前
- [ ] 标签样式区分明显，易于识别
