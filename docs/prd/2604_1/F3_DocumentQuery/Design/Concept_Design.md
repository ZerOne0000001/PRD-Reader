# 概念设计 (Concept Design) - F2 文档查询功能

## 1. 设计方向选择

### 1.1 候选方案对比

| 方案 | 描述 | 优点 | 缺点 |
| :--- | :--- | :--- | :--- |
| **方案 A：极简列表式** | 搜索结果页面采用纯列表展示，无侧边筛选 | 界面简洁，工作量小，用户一目了然 | 当结果跨多个仓库时，缺乏快速筛选手段 |
| **方案 B：左侧筛选 + 右侧列表** | 搜索结果页左侧增加仓库筛选面板，右侧主区域展示结果列表 | 支持按仓库精准筛选，适合白名单仓库较多的场景 | 界面复杂度增加，筛选面板占用横向空间 |

### 1.2 方案决策

**✅ 最终选择：方案 B - 左侧筛选 + 右侧列表**

**选择理由**：
1. PRD-Reader 定位是多仓库文档管理平台，白名单内可能有多个仓库
2. 用户在不同仓库间切换查找文档是核心场景
3. 左侧筛选面板可以让用户快速缩小搜索范围，提升效率
4. 筛选面板采用可折叠设计，不会过多占用空间

## 2. 概念方案详细设计

### 2.1 页面整体布局

```
┌─────────────────────────────────────────────────────────────┐
│  顶部导航栏 (Header)                                         │
│  ┌─────────────────────────────────────────────────────────┐│
│  │ [← 返回]     搜索框 [关键词]     [搜索按钮]               ││
│  └─────────────────────────────────────────────────────────┘│
├────────────┬────────────────────────────────────────────────┤
│ 左侧筛选面板 │  搜索结果列表区域                              │
│ (Sidebar)  │                                               │
│            │  共找到 X 个结果                    第 1/5 页   │
│ ☑ 仓库 A   │  ┌────────────────────────────────────────┐   │
│ ☑ 仓库 B   │  │ 📄 filename.md                    内容匹配 │   │
│ ☐ 仓库 C   │  │ 仓库A / path/to/file                  │   │
│            │  │ ...这是搜索关键词的上下文预览片段...     │   │
│ ─────────  │  └────────────────────────────────────────┘   │
│ 匹配类型    │  ┌────────────────────────────────────────┐   │
│ ☑ 文件名    │  │ 📄 another.md                    文件名匹配│   │
│ ☑ 内容      │  │ 仓库B / other/path                    │   │
│            │  │ filename.md                             │   │
│            │  └────────────────────────────────────────┘   │
└────────────┴────────────────────────────────────────────────┘
```

### 2.2 交互流程概念

1. **用户输入关键词** → 点击搜索按钮
2. **跳转搜索结果页** → 展示加载状态
3. **加载完成** → 左侧面板显示所有仓库（默认全选），右侧显示结果列表
4. **用户勾选/取消仓库** → 实时过滤结果列表
5. **用户切换匹配类型** → 实时过滤结果列表（文件名/内容）
6. **用户点击分页** → 加载对应页码结果
7. **用户点击结果** → 新标签页打开 Reader 并加载该文件

### 2.3 核心组件抽象

| 组件 | 职责 | 状态 |
| :--- | :--- | :--- |
| `SearchBar` | 搜索框输入 + 触发搜索 | 输入态、加载态、禁用态 |
| `FilterSidebar` | 仓库多选 + 匹配类型切换 | 展开态、收起态 |
| `SearchResultList` | 结果列表展示 | 加载态、空结果态、正常态、错误态 |
| `SearchResultItem` | 单条结果展示 | 默认态、悬停态 |
| `Pagination` | 分页控件 | 正常态、禁用态（首页/末页） |

## 3. 技术概念

### 3.1 前端路由设计

| 路由 | 页面 | 说明 |
| :--- | :--- | :--- |
| `/reader` | Reader.tsx | 阅读器主页（现有） |
| `/search?q=关键词` | Search.tsx (新建) | 搜索结果页面 |

### 3.2 状态管理概念

| Store | 职责 |
| :--- | :--- |
| `useSearchStore` (新建) | 管理搜索关键词、搜索结果列表、当前页码、筛选状态（仓库、匹配类型）、加载状态 |

### 3.3 API 设计

| 接口 | 方法 | 说明 |
| :--- | :--- | :--- |
| `/api/gitlab/search` | GET | 搜索接口，调用 GitLab Search API 并过滤白名单仓库 |

**Query 参数**：
- `q`: 搜索关键词
- `scope`: 搜索范围（blobs=内容, projects=项目名）
- `page`: 页码
- `per_page`: 每页数量

## 4. 设计原则

1. **渐进式披露**：搜索结果页默认收起左侧筛选面板，用户鼠标悬停时展开
2. **即时反馈**：筛选条件变更时，结果列表实时更新（防抖处理）
3. **状态可见**：加载中、空结果、错误等状态都有明确的视觉反馈
4. **关键词高亮**：搜索结果中的关键词使用 `<mark>` 标签高亮
5. **防泄露**：后端必须过滤结果，仅返回白名单仓库内的数据
