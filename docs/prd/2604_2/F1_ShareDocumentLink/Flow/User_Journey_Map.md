# 用户旅程地图 (User Journey Map)

| 阶段 | 发现与分享 | 复制链接 | 直达与加载 | 异常处理 (分支) |
| :--- | :--- | :--- | :--- | :--- |
| **用户行为** | PM 在左侧树中找到需要评审的 PRD（如 `.md` 文件）并点击阅读。 | 鼠标悬停在左侧节点上，点击原有的“分享” Icon，在弹窗中选择“复制系统分享链接”。 | 开发在微信/企微收到链接后点击，浏览器打开该链接。 | 开发点击了一个已废弃或重命名的文件分享链接。 |
| **系统响应** | 左侧树高亮选中节点，右侧渲染内容；同时，浏览器地址栏 URL 实时更新为 `?repo=xxx&file=xxx`。 | 弹出包含“Git 地址”和“系统链接”的选项菜单；点击后调用剪贴板 API 写入完整的当前页面 URL。 | 系统初始化加载：优先解析 URL 参数 -> 验证白名单 -> 异步拉取仓库树 -> 自动逐层展开至目标节点并高亮 -> 右侧请求并渲染文件内容。 | 系统初始化解析参数，在请求文件内容（或树节点）时收到后端 404/403 响应，右侧阅读区渲染特定的缺省状态页面。 |
| **触点 (Touchpoints)** | 左侧文件树节点、浏览器地址栏 | 悬浮分享 Icon、选项弹窗 (Popover/Modal)、剪贴板成功提示 (Toast) | 浏览器加载进度条、系统 Loading 骨架屏、左侧文件树、右侧阅读区 | 浏览器、右侧阅读区特定的 404 缺省视图 |
| **痛点/机会点** | **机会**：将 URL 状态同步作为一种无感知的底层能力，用户直接复制地址栏也能达到目的，无需强迫使用界面按钮。 | **痛点解决**：统一分享入口，避免新增按钮导致左侧树结构拥挤；提供明确的选项，满足不同场景需求。 | **痛点解决**：彻底消除“给我一个链接，我还要自己去找”的沟通成本，实现“开箱即用”的丝滑体验。 | **痛点解决**：优雅地处理链接失效问题，避免系统“死加载”或白屏，提升产品的鲁棒性。 |
