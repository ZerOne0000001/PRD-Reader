# 功能清单 (Feature List)

## 1. 概览
本需求涉及对现有核心阅读器模块（`F02_Reader`）的改造，主要分为四个维度的功能点：URL 状态同步机制、左侧文件树交互改造（分享入口与下钻能力）、右侧阅读区异常处理渲染。

## 2. 功能清单表

| 一级模块 | 二级功能 | 功能描述 | 优先级 | 备注说明 |
| :--- | :--- | :--- | :--- | :--- |
| **URL 状态同步** | **动态 URL 更新** | 当用户在左侧文件树中点击任何有效节点（仓库、文件夹、文件）时，浏览器的地址栏 URL 必须实时且无刷新地更新查询参数（如 `?repo=123&file=docs/prd.md`）。 | P0 | 核心需求，基于 `history.replaceState` 或 `react-router` 实现。 |
| **左侧文件树** | **悬浮分享菜单 (Popover)** | 改造原有的“复制 Git 地址” Icon。鼠标悬停在节点上点击该 Icon 时，弹出一个小型菜单，提供“复制 Git 地址”和“复制系统分享链接”两个选项。 | P0 | 交互重构点。 |
| | **复制系统链接交互** | 点击“复制系统分享链接”后，系统拼装包含当前域名、路由和 `repo`、`file` 参数的完整 URL，写入系统剪贴板，并在界面上给予轻量级的成功提示（Toast 或 Icon 状态变化）。 | P0 | |
| | **初始加载“下钻”定位** | 页面初始化加载时，若 URL 携带参数，系统在内存构建完完整的仓库目录树后，需反向查找到该 `file` 路径所在的层级。 | P0 | 依赖状态机 `readerStore.ts` 算法的扩展。 |
| | **文件夹下钻渲染** | 如果分享链接指向的是一个**文件夹**，左侧树直接以该文件夹为根节点进行渲染（下钻模式），并高亮该文件夹节点。 | P0 | 强需求约束。 |
| | **文件下钻渲染** | 如果分享链接指向的是一个**具体文件**，左侧树直接以**该文件所在的父文件夹**为根节点进行渲染（下钻模式），并高亮该文件节点。 | P0 | 强需求约束。 |
| **主阅读区渲染** | **缺省页：无权限/未配置仓库** | 当分享链接中的 `repo` 参数不在当前系统的配置白名单中时，在阅读区原位渲染特定的 404 状态页，提示“仓库无权限或未配置”。 | P1 | 优雅的异常处理。 |
| | **缺省页：文件/路径不存在** | 当分享链接指向的 `file` 路径在树结构中无法找到时，在阅读区原位渲染特定的 404 状态页，提示“文件不存在或已被删除”。 | P1 | 优雅的异常处理。 |
| | **加载状态骨架屏** | 在系统根据分享链接进行树结构构建和内容拉取期间，阅读区应有合适的 Loading 状态（如骨架屏或加载动画）。 | P2 | 提升大文件仓库的体验。 |
