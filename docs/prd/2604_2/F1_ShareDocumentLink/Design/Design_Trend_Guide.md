# 设计倾向指南 (Design Trend Guide)

## 1. 核心调性 (Tone)
* **风格定义**：极简 Notion 风 (Seamless Integration)
* **核心目标**：新增的 UI 元素（Popover 气泡菜单、404 缺省状态页、Loading 骨架屏）必须与现有的系统风格无缝融合，**绝对禁止**使用任何突兀的渐变、圆角过大的按钮或花哨的阴影。强调界面的纯净和内容的优先级。

## 2. Typography (排版规则)
* **字体栈**：优先使用系统默认无衬线字体，或类似于 Inter 的无衬线字体（但不可泛滥使用加粗）。
* **字号与层级**：
  - Popover 菜单文字：`13px` 或 `14px`，保持克制，不可喧宾夺主。
  - 404 缺省页主标题：`24px`，加粗，深灰色 (`#111827`)。
  - 404 缺省页副标题：`14px`，常规体，浅灰色 (`#6B7280`)。

## 3. Color & Theme (色彩与主题)
* **背景色**：
  - Popover 背景：纯白 `#FFFFFF`，带有极淡的边框 `#E5E7EB`。
  - 404 缺省页背景：跟随右侧主阅读区的底色（通常是纯白）。
* **交互色**：
  - Hover 状态（菜单项）：极淡的灰色背景 `#F3F4F6`，文字颜色不变或略微加深。
  - 选中/成功反馈（如链接复制成功）：使用现有的 `[var(--accent-blue)]` 或轻微的绿色（用于成功打勾 Icon），不可使用高饱和度色块。
* **边框与分隔线**：极细（`1px`），极浅（`#F3F4F6` 或 `#E5E7EB`）。

## 4. Spatial Composition (空间排版与形状)
* **圆角 (Border Radius)**：
  - Popover 容器：圆角控制在 `8px` 以内（`rounded-lg`）。
  - 菜单项：圆角控制在 `4px` 左右（`rounded-md`）。
* **阴影 (Shadow)**：
  - Popover 必须带有非常克制的环境阴影（如 Tailwind 的 `shadow-md` 或 `shadow-sm`），**绝对禁止**使用扩散面积极大的厚重阴影。
* **布局**：
  - Popover 应紧贴触发它的分享 Icon，使用绝对定位或相对定位，并留有微小的间隙（`4px` - `8px`）。
  - 404 缺省页：在主阅读区绝对居中，插画/Icon、主标题、副标题、操作按钮自上而下居中对齐，间距宽松，留白充足。

## 5. Anti-AI Slop Rules (反 AI 塑料感红线)
* **禁止**使用大面积的蓝色/紫色渐变。
* **禁止**在 404 页面使用过于复杂的 3D 插画，请使用简洁的 SVG 线性 Icon（如断裂的文档、锁等，类似 Lucide Icons）。
* **禁止**在 Popover 中使用庞大的图标或过度的留白，菜单应该紧凑、干练，像原生操作系统的右键菜单一样。
