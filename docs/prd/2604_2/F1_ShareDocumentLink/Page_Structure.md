# 页面结构 (Page Structure)

## 1. 全局导航与结构
本次迭代**不新增**新的 HTML/页面级路由文件，所有的改造均在现有的阅读器核心组件中完成。

- **目标页面**：`Reader.tsx` (主阅读区页面，通常映射为应用首页 `/` 或 `/reader`)

## 2. 功能分区拆解

### 2.1 状态信息区 (全局状态同步)
- **隐形核心逻辑**：
  - **加载监听器**：页面 Mount 时（`useEffect`），读取浏览器的 Query Params (`repo`, `file`)。
  - **状态机调度器**：若存在参数，则通知 `readerStore` 执行“下钻加载 (Drill-down)”及“文件拉取”操作。
  - **URL 同步器**：监听 `readerStore` 中 `currentFile` 或选中节点的变化，静默调用 `setSearchParams` 实时更新地址栏的 `repo` 和 `file` 参数。

### 2.2 左侧抽屉式文件树 (Drawer Tree)
- **节点组件改造 (`TreeNode`)**：
  - **交互触发点**：鼠标悬停在任意有效节点（云朵/文件夹/文档/图片 Icon 所在行）时，显示原有的“分享”或“复制” Icon。
  - **分享选项菜单 (Popover/Dropdown)**：
    - 点击分享 Icon 后，**不在原位**直接复制，而是浮出一个气泡菜单。
    - **菜单项 1**：【复制 Git 原地址】（保留原有逻辑，仅针对支持的平台节点显示）。
    - **菜单项 2**：【复制系统分享链接】（核心新增逻辑）。
  - **操作反馈 (Toast/Icon State)**：
    - 点击菜单项后，调用剪贴板 API 将拼装好的 URL (`window.location.origin + window.location.pathname + ?repo=xxx&file=yyy`) 写入剪贴板。
    - 成功后，菜单消失，原分享 Icon 变为绿色的打勾状态，停留 2 秒后恢复原状（或弹出全局 Toast 提示“系统链接已复制”）。
- **树结构下钻渲染 (Drill-down Render)**：
  - 若系统处于分享链接的“下钻”模式，左侧树的根节点不再是全局的所有仓库。
  - **如果是分享文件夹**：树的顶层直接显示该文件夹的名称，其下展开子节点。
  - **如果是分享文件**：树的顶层显示该文件所在的**父文件夹**名称，其下展开该文件及同级兄弟节点。
  - **顶部导航**：在下钻模式下，抽屉面板顶部需提供一个明确的“返回上级”或“返回根目录”的按钮，以便用户退出分享的局部视图，浏览完整的仓库列表。

### 2.3 右侧主阅读区 (Main Content Area)
- **内容渲染容器 (`renderContent`) 改造**：
  - **Loading 状态**：在根据 URL 参数异步拉取庞大的仓库树结构，或请求文件 Raw 内容时，居中展示加载骨架屏或 Loading Spinner（“正在定位分享文档...”）。
  - **正常渲染**：保持原有的 Markdown (`.md`)、HTML (`.html`) 或图片组件渲染逻辑不变。
  - **404 缺省状态拦截 (新增)**：
    - **场景 1：仓库未授权**：当 URL 中的 `repo` 不在配置白名单内，或者后端返回 403/404 时。
      - 居中展示特定的插画/Icon（如一把锁或一个空箱子）。
      - 主标题：“仓库无权限或未配置”。
      - 副标题：“您访问的分享链接指向了一个未被系统授权的仓库。请联系管理员添加白名单。”
    - **场景 2：文件/路径不存在**：当 URL 中的 `file` 路径在树结构中反向查找失败，或拉取内容返回 404 时。
      - 居中展示特定的插画/Icon（如一张撕裂的纸或放大镜）。
      - 主标题：“文件不存在或已被删除”。
      - 副标题：“您访问的文档路径在此仓库中无法找到。它可能已被重命名、移动或删除。”
    - **操作指引**：在缺省页下方提供一个“返回首页”或“浏览其他可见仓库”的按钮，点击后清除 URL 参数，恢复默认的全局树加载状态。
