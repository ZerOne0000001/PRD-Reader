# 需求背景 (Requirement Background)

## 1. 业务目标与背景
* **当前痛点**：目前 PRD-Reader 系统在团队协作（如 PM 分享文档给开发、测试）时，只能分享系统的根域名。接收方打开后，需要自行在左侧庞大的文件树中逐层寻找目标文件，这不仅沟通成本极高，而且非常容易找错文档版本或路径。
* **业务目标**：实现系统的深度链接（Deep Linking）与便捷分享机制。使系统的 URL 能够实时反映当前正在阅读的仓库和文件（或文件夹）路径；同时提供显式的分享入口，让分享者一键获取链接，接收者点击链接后能“开箱即用”，系统自动定位并渲染目标内容。

## 2. 用户画像 (User Personas)

| 角色 | 特征 | 核心痛点 | 行为限制/偏好 |
| :--- | :--- | :--- | :--- |
| **产品经理 (分享者)** | 频繁查阅和评审需求文档，需要将定稿或需讨论的文档精准同步给项目组其他成员。 | 每次都要截图说明文件路径，或口头指导开发去哪里找文档，效率极低。 | 偏好在阅读文档的同时，能顺手、直观地获取当前文档的直达链接；偏好系统提供清晰的分享选项。 |
| **开发/测试 (接收者)** | 根据需求文档进行编码或编写测试用例，强依赖文档的准确性。 | 收到一个根域名后，要在复杂的树结构里找很久，有时还会看错同名文件。 | 偏好“所见即所得”，点击群里的链接就能直接在浏览器中看到目标文档的内容，无需任何额外操作。 |

## 3. 核心场景 (Core Scenarios)

| 场景名称 | 关联用户画像 | 触发条件与环境 | 当前痛点 | 核心动作 | 期望结果 | 潜在异常 |
| :--- | :--- | :--- | :--- | :--- | :--- | :--- |
| **生成与复制分享链接** | 产品经理 (分享者) | 在系统左侧树中找到并点击了某个具体的文件或文件夹。 | 缺乏直接获取当前文档系统内链接的便捷入口。 | 1. 浏览器 URL 实时更新为包含参数的路径。<br>2. 点击左侧树节点原有的“分享 git 地址” icon，唤起选项弹窗。<br>3. 在弹窗中选择复制“系统内部分享链接”。 | 成功将带有 `repo` 和 `file` 参数的系统链接复制到剪贴板，并给予轻量级的成功提示。 | 浏览器不支持剪贴板 API；分享的节点是一个无法预览的特殊格式。 |
| **通过分享链接直达文档** | 开发/测试 (接收者) | 在工作群中点击了分享者发来的带有参数的系统链接。 | 只能打开系统首页，必须手动寻找文件。 | 1. 浏览器打开链接，系统解析 URL 参数。<br>2. 系统静默校验仓库权限和文件存在性。<br>3. 自动在左侧树展开对应层级并高亮该节点。<br>4. 在右侧主阅读区直接渲染该文件内容。 | 完美复现分享者当时的阅读状态，实现“开箱即用”。 | 链接指向的文件已被删除；链接指向的仓库不在当前系统的白名单内；网络延迟导致树结构尚未加载完成。 |
| **访问无效或无权限链接** | 开发/测试 (接收者) | 点击了一个过期或错误的分享链接。 | 系统可能卡死或表现异常，无明确指引。 | 系统解析参数后，在尝试拉取文件信息时遭遇错误。 | 右侧主阅读区展示一个特定的 404 缺省状态页面（如“文件不存在或无权限”），左侧树保持默认加载状态。 | 后端接口返回的错误类型不明确，导致前端无法精准区分是 404 还是 403。 |
