# 系统调研与上下文分析 (System Survey)

## 1. 原有系统核心逻辑
本项目是一个基于 React + Zustand 构建的纯前端单页应用 (SPA)。核心页面 `Reader.tsx` (阅读区) 负责展示左侧文件树和右侧主阅读区。
其状态管理 `store/readerStore.ts` 具有以下特点：
- **扁平转嵌套树**：后端接口返回的通常是包含完整 `path` 的扁平化文件列表，前端通过 `buildNestedTree` 算法在内存中一次性组装为带有 `children` 的完整树结构。
- **当前文件状态**：通过 `currentFile` 对象维护右侧主阅读区的渲染内容和类型（`md`, `html`, `image` 等）。
- **路由无状态化**：目前系统的阅读状态是纯内存维护的，并未与浏览器的 URL 地址（URL Query 或 Hash）进行深度双向绑定。

## 2. 受影响的现有页面与模块
本次需求（分享文档链接）是对**现有系统的迭代/修改**，核心影响以下几个部分：
1. **`src/pages/Reader.tsx`**：
   - 核心布局的初始化钩子 (`useEffect`) 需要拦截解析 URL 中的 `repo` 和 `file` 参数。
   - 左侧 `TreeNode` 树节点组件上的“复制 Git 地址” Icon 需要改为触发自定义的弹出层 (Popover/Dropdown)，以供用户选择复制类型。
   - 右侧主阅读区需要增加对 404 异常状态的渲染支持。
2. **`src/store/readerStore.ts`**：
   - 现有的 `buildNestedTree` 及树节点数据结构需要增加辅助方法，以支持根据 `filePath` 在完整的树中**反向查找到该节点所在的层级路径**。
   - 新增“进入目录（下钻）”的逻辑：当分享链接指向某个文件夹（或该文件夹下的文件）时，左侧树应当**直接以该文件夹为根节点进行渲染**，而不是展开庞大的全局树。
3. **路由组件**：
   - React Router 的配置可能需要适配带有 Query 参数的导航。

## 3. 约束与依赖
- **URL 参数结构**：必须使用标准的查询字符串形式（如 `?repo=123&file=docs/prd.md`），以保证可读性和后续扩展性。
- **“下钻”加载策略**：用户明确要求：分享文件夹时，直接进入该文件夹（下钻模式）；分享文件时，直接进入该文件所在的文件夹并打开文件。这意味着状态机必须具备**局部树根节点切换（Drill-down）**的能力。
- **404 组件级拦截**：发生错误时，不触发全局路由跳转，而是由 `Reader.tsx` 捕获异常并在主阅读区原位渲染 404 缺省页，保证侧边栏仓库列表的稳定。
