# 用户旅程地图 (User Journey Map)

**主题**: 在交互式 PRD 中独立全屏查看原型图
**目标用户**: 研发/测试/设计师 (Reader)
**场景设定**: 用户正在阅读 `Interactive_PRD.html`，需要对照大图原型进行开发或走查。

| 阶段 | 发现问题与需求产生 | 交互入口发现 | 核心动作执行 | 获得期望结果与收尾 |
| :--- | :--- | :--- | :--- | :--- |
| **用户行为** | 发现当前交互式 PRD 右侧的原型图较小或需要横向滚动，想在全屏显示器上查看。 | 视线移动到右侧 iframe 区域寻找独立查看的方法，发现右上角的悬浮外链图标。 | 鼠标悬停在图标上看到“在新标签页打开原型”的提示，并点击该图标。 | 在新弹出的浏览器标签页中，查看到全尺寸的原型页面，并继续阅读。 |
| **触点** | `Interactive_PRD.html` 右侧 iframe 容器 | iframe 容器右上角悬浮的“新标签页打开” SVG 图标。 | 图标悬停态、浏览器默认的 `window.open` 行为。 | 新的浏览器标签页，URL 对应 `src#hash`。 |
| **情绪/心理** | 😑 感觉被狭窄的分栏局限了视野，看不清细节。 | 🧐 注意到右上角有交互按钮，试图了解其功能。 | 🙂 点击时期待马上能在新标签页打开大图。 | 😃 获得全屏体验，可以拖拽到副屏上对比着写代码了。 |
| **系统行为** | 系统保持现有的左右分栏联动状态，iframe 正常展示。 | 系统展示图标，响应 hover 事件弹出 tooltip。 | JavaScript 捕获点击事件，获取 iframe 当前实际的 `src` 属性，调用 `window.open(src, '_blank')`。 | 浏览器新标签页渲染 HTML 原型文件，附带当前的 URL Hash 供原型内部响应。 |
| **机会点 / 痛点改善** | 避免用户必须去文件资源管理器中手动查找原型的相对路径文件。 | 图标的视觉设计需要显眼但不过于突兀，不遮挡原型的核心内容。可采用半透明或毛玻璃效果底色。 | 必须确保获取到的 `src` 是最新的（包括左侧 TOC 联动带来的 hash 变化），避免打开的还是首页或旧位置。 | 提升了阅读和开发效率，打通了“文档 -> 全屏走查”的无缝切换。 |
