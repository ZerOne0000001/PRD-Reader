# 用户故事 (User Stories)

## Epic 1: 交互式 PRD 原型查看体验优化

### Story 1.1: 作为读者，我希望能在新标签页中打开当前正在查看的原型，以便更清晰地查看大图和细节。
**As a** 研发/测试/设计师 (Reader)
**I want to** 在交互式 PRD 的右侧原型预览区右上角，看到并点击一个“在新标签页打开”的图标，从而在独立的新浏览器标签页中打开当前的原型页面。
**So that** 我可以摆脱左右分栏的宽度限制，在全屏模式下无干扰地查看设计细节，或者将独立窗口拖拽到另一块显示器上进行对照开发。

#### 验收标准 (Acceptance Criteria):
1. **图标可见性**：在 `Interactive_PRD.html` 的右侧 iframe 容器的右上角，必须固定显示一个清晰的“外部链接 (External Link)”图标（例如基于 SVG 的标准外链图标）。
2. **悬停提示**：鼠标悬停在图标上时，应显示类似 `在新标签页打开原型` 或 `Open prototype in new tab` 的 Tooltip 提示。
3. **点击行为**：用户点击图标后，通过 `window.open` 行为在新的浏览器标签页中打开当前 iframe 所加载的 `src` URL。
4. **状态同步**：如果在当前交互式 PRD 中，原型页面已经跟随左侧目录的点击切换了 hash（例如 `index.html#section-a`），点击新标签打开时，必须携带该 hash 状态，保证新页面也能定位到相应位置。
5. **视觉风格**：图标的样式（颜色、大小、悬停态等）应符合蓝图 (Blueprint) 中定义的 Soft & Pastel 风格，且不应遮挡原型本身的核心交互区域（可以使用半透明背景或在悬停时加深）。
