# 系统调研报告 (System Survey)

## 1. 调研目标
分析在 `interactive-prd-builder` 产出的交互式 PRD (`Interactive_PRD.html`) 中增加“在新标签页打开原型”功能的可行性及系统影响。

## 2. 原有系统核心逻辑
目前的 `Interactive_PRD.html` 是一个单文件应用（或静态 HTML 组合），核心逻辑为：
*   **布局结构**：左侧为 TOC（目录/PRD 文档区域），右侧为一个 `<iframe id="prototype-frame">` 容器，用于加载各个原型的 HTML 文件。
*   **状态同步**：通过外层页面的 URL Hash（如 `#F1_Home/index.html`）来控制右侧 iframe 加载对应的页面，并在原型内部通过 `hashchange` 事件实现组件状态的联动切换。

## 3. 受影响的现有页面/组件
*   **外层容器 (`Interactive_PRD.html` 模板)**：需要在右侧 iframe 容器（通常是一个 `div` 包裹着 `iframe`）的相对定位节点上，增加一个绝对定位的 `<button>` 或 `<a>` 图标元素。
*   **样式文件/内联 CSS**：需要增加该悬浮图标的样式定义（常驻半透明，悬停时加深，位于右上角）。
*   **脚本逻辑**：需要增加图标的点击事件监听器。

## 4. 约束与依赖
*   **跨域/同源策略**：由于是在本地通过文件协议 (`file://`) 打开，或者在静态服务器下同源打开，读取 iframe 内容可能会遇到限制。因此，获取 URL 的方式应尽量依赖外层可控的状态。
*   **视觉冲突防护**：为了防止该图标遮挡原型页面本身的右上角内容（如弹窗的关闭按钮等），已确认采用**常驻半透明，悬停时恢复不透明度**的设计方案，并确保其 `z-index` 在外层容器中处于顶层。
*   **联动性约束**：必须确保“新标签页”打开时的 URL，与当前用户在左侧菜单点击后产生的最新状态完全一致。
