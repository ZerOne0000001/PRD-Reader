# 需求背景与目标 (Requirement Background)

## 1. 业务目标与背景
**背景说明**：在目前通过 `interactive-prd-builder` 技能生成的交互式 PRD (`Interactive_PRD.html`) 中，采用了经典的左右分栏布局（左侧导航/文档，右侧 iframe 嵌入原型 HTML）。虽然这种布局方便图文对照，但当原型页面设计较为复杂、尺寸较大时，受限于右侧区域的宽度，用户难以看清原型细节。
**业务目标**：通过在 iframe 区域增加一个“新标签页打开”的功能入口，为用户提供一种快捷跳出当前分栏束缚、在全屏或独立标签页中查看原型细节的途径，从而提升整体的阅读与开发走查体验。

## 2. 用户画像 (User Personas)
| 角色名称 | 特征描述 | 核心痛点 | 行为限制与偏好 |
| :--- | :--- | :--- | :--- |
| 研发/测试 (Reader) | 需要高频对照 PRD 规则和原型图进行开发与验收走查。 | 左右分栏布局下右侧展示空间有限，对于复杂的页面结构或大图，频繁滚动十分低效且容易看漏细节。 | 偏好在多显示器环境下工作，喜欢将原型图单独拖拽到一个屏幕全屏查看，同时在另一个屏幕查看代码或文字规则。 |
| 产品经理 (PM) | 负责向团队演示或核对设计细节。 | 演示时如果只在小窗口中展示，会议室投影仪上看不清。 | 偏好有一键最大化或新开窗口的快捷操作，避免手动去文件目录里找对应的 HTML 文件。 |

## 3. 核心场景 (Core Scenarios)
| 场景名称 | 关联用户画像 | 触发条件与环境 | 当前痛点 | 核心动作 | 期望结果 | 潜在异常 |
| :--- | :--- | :--- | :--- | :--- | :--- | :--- |
| 独立全屏查看原型图 | 研发/测试、PM | 在日常需求评审或开发阶段，用户正在阅读 `Interactive_PRD.html`，发现右侧原型信息密度过高，需要全屏对照。 | 必须手动去项目目录下寻找对应的原型 HTML 文件并双击在浏览器中打开，路径深且繁琐。 | 鼠标悬停至右侧 iframe 区域右上角，点击“在新标签页打开” (Open in new tab) 的悬浮图标。 | 浏览器立即弹出一个新标签页，并加载当前 iframe 中正在展示的原型 HTML 页面（携带对应的锚点状态）。 | 浏览器的弹出窗口拦截机制可能会拦截（但在用户主动点击触发时一般不会被拦截）。 |
