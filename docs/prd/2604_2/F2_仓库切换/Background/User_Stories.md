# 用户故事：仓库切换与独立列表页 (F2_仓库切换)

---

### **故事一：独立仓库列表页访问**
**作为** 一名经常需要在不同项目中查阅文档的产品经理
**我希望** 在不指定任何文件链接的情况下打开系统时，能自动进入一个独立的“仓库列表页”
**以便于** 我能快速找到自己当前需要的那个项目/仓库。

**验收标准 (Acceptance Criteria):**
1.  **自动导流**：如果用户访问的 URL 中不包含 `?repo=` 和 `?file=` 等明确的文档指向参数，系统必须自动重定向或渲染独立的“仓库列表页”。
2.  **内容展示**：仓库列表页应以卡片或列表的形式展示当前系统配置的“白名单仓库”（包含仓库名、图标或简短描述）。
3.  **交互跳转**：点击某个仓库卡片，系统应跳转进入阅读器页面，并在左侧文件树中**仅展示该仓库的内容**，右侧默认展示该仓库的介绍文件（如 README.md）或欢迎页。

---

### **故事二：阅读器内快速切换仓库**
**作为** 一名正在沉浸式阅读 API 文档的开发者
**我希望** 能够在阅读器页面的左上角点击当前的仓库名称，弹出一个仓库列表，并能直接选中切换
**以便于** 我不需要退回到列表页，就能快速在多个仓库间切换上下文。

**验收标准 (Acceptance Criteria):**
1.  **入口位置**：在阅读器界面的左上角（紧邻或替代原本的全局标题位置），必须显示当前正在阅读的仓库名称，并且样式上暗示其可点击（如带有向下箭头）。
2.  **下拉交互**：点击该名称，必须弹出一个下拉列表，展示所有可用的白名单仓库。
3.  **切换行为**：当用户在下拉列表中点击另一个仓库时，阅读器页面的左侧文件树和右侧内容区必须立即刷新为目标仓库的内容，且 URL 的 `?repo=` 参数需同步更新。
4.  **状态提示**：切换仓库时，如果加载时间较长，必须有合理的 Loading 骨架屏或提示。

---

### **故事三：从阅读器返回仓库列表**
**作为** 一名想要纵览所有项目的产品经理
**我希望** 在阅读器页面的左上角仓库下拉列表中，能够提供一个“返回仓库列表”的选项
**以便于** 我能以最符合直觉的连贯方式退回到宏观视角。

**验收标准 (Acceptance Criteria):**
1.  **选项位置**：在阅读器左上角的下拉仓库列表中，必须包含一个显眼的“返回选择仓库页面”或“所有仓库”的选项（通常位于列表的顶部或底部）。
2.  **路由行为**：点击该选项后，系统必须跳转回独立的“仓库列表页”，并且不带任何 `?repo=` 或 `?file=` 参数。
3.  **浏览器历史**：该操作应该通过正常的路由跳转实现，确保用户依然可以使用浏览器的“后退”按钮回到之前的阅读器状态。
