# 需求背景：仓库切换与独立列表页 (F2_仓库切换)

## 1. 业务目标与背景
**现状问题**：在当前的 PRD-Reader 系统中，多个代码/文档仓库的文件树会被堆积在一起显示。当配置的仓库数量增多时，左侧文件树会显得过于拥挤和混乱，导致产品经理或开发者在寻找特定项目的文件时，需要耗费大量时间进行滚动和辨识。
**业务目标**：
1.  **降低认知负荷**：通过将仓库选择从具体文件阅读中剥离出来，提供一个清晰的“独立仓库列表页”，让用户在进入具体文档前能够快速定位目标项目。
2.  **提升跨项目查阅效率**：在阅读器内（Reader）的左上角增加仓库切换的快捷入口，支持在不丢失阅读沉浸感的前提下，通过下拉列表快速切换仓库或返回仓库列表页。

## 2. 用户画像 (User Personas)

| 角色 | 特征 | 核心痛点 | 行为限制/偏好 |
| :--- | :--- | :--- | :--- |
| **产品经理 (PM)** | 经常需要在多个产品线（对应不同仓库）的 PRD 和原型之间进行查阅和对比。 | 仓库混杂在一起时，很难快速找到自己负责或当前需要查阅的那个项目的文档。 | 偏好清晰的视觉层级和直接的导航，希望打开系统第一眼就能看到项目（仓库）卡片。 |
| **开发者 (Dev)** | 关注具体的 API 契约和技术实现文档，偶尔需要跨前后端仓库核对数据模型。 | 在左侧文件树中滚动查找跨仓库的文件时，由于文件夹层级过深，极易迷失。 | 习惯于IDE中的“切换 Project/Workspace”概念，希望能在阅读器内快速切换上下文。 |

## 3. 核心场景 (Core Scenarios)

| 场景名称 | 关联用户画像 | 触发条件与环境 | 当前痛点 | 核心动作 | 期望结果 | 潜在异常 |
| :--- | :--- | :--- | :--- | :--- | :--- | :--- |
| **场景一：默认进入仓库导航** | 产品经理、开发者 | 用户直接输入系统域名，未携带任何具体的文件或文件夹参数（URL 中无 `?repo=` 和 `?file=`）。 | 默认进入庞杂的混合文件树，不知从何看起。 | 浏览独立的仓库列表页面，点击所需的仓库卡片。 | 系统自动进入阅读器页面，左侧文件树仅展示选中仓库的内容。 | 配置的白名单仓库为空，导致列表页无数据。 |
| **场景二：阅读中快捷切换仓库** | 产品经理、开发者 | 用户正在阅读 A 仓库的某篇文档，突然需要查看 B 仓库的文档。 | 需要手动修改 URL 或返回首页重新在一大堆文件夹中寻找 B 仓库。 | 点击左上角当前仓库名称，在下拉列表中选择 B 仓库。 | 页面刷新，左侧文件树变为 B 仓库，右侧展示 B 仓库的默认引导页（如 README）。 | 下拉列表加载失败，或选中的仓库因权限变更无法访问。 |
| **场景三：从阅读器返回列表页** | 产品经理 | 用户看完了当前仓库的所有文档，想要回到宏观视角寻找其他项目。 | 只能通过浏览器的“后退”按钮盲目返回，体验不连贯。 | 点击左上角仓库名称弹出的下拉列表中，点击“返回仓库列表”按钮。 | 退出当前阅读器视图，返回到全局的独立仓库列表页。 | 浏览器历史栈混乱导致返回行为不符合预期（应确保使用路由跳转）。 |
