# 页面结构 (Page_Structure) - F2_仓库切换

## 1. 全局导航与结构
本需求涉及两个主要视图的改动：
1.  **独立仓库列表页**：作为系统的根路由入口（`home.html`，对应现有系统的首页逻辑分支）。
2.  **阅读器页 (Reader)**：现有的主工作区（`index.html`，当带有 `?repo=` 参数时进入）。

整体采用桌面端响应式设计（响应式折叠，隐藏滚动条），延续现有的极简清新风（Notion 风格）。

---

## 2. 独立仓库列表页 (`home.html` 或重构后的根路由)

**2.1 页面容器 (Page Container)**
*   **满屏居中布局**：整个页面内容垂直水平居中或带有较大顶部留白（类似搜索主页或 Dashboard 初始页）。
*   **全局操作入口**：右上角提供“后台设置”的快捷入口按钮。

**2.2 头部信息区 (Header Area)**
*   **Logo/系统名称**：PRD-Reader 的系统 Logo 或系统名称。
*   **欢迎语/提示文案**：例如“请选择要阅读的仓库”。

**2.3 仓库卡片网格区 (Repository Grid Area)**
*   **网格布局**：使用 CSS Grid 布局，自动响应式换行（缩小卡片最小宽度，使得桌面端能展示 3-5 列，提高空间利用率）。
*   **仓库卡片组件 (Repository Card)**：
    *   **图标占位**：左侧或顶部展示一个由仓库名称首字母生成的彩色圆形占位符（Avatar/Icon），增加辨识度。
    *   **文本信息**：
        *   **仓库名称 (name)**：醒目的仓库名称。
        *   **仓库路径 (path)**：全路径或命名空间（如 `group/project-name`）。
        *   **默认分支 (default_branch)**：通过小标签形式展示（如 `main`）。
    *   **交互状态**：鼠标悬浮（Hover）时卡片有轻微的上浮动画或阴影加深（Elevation），鼠标指针变为手型。
    *   **点击动作**：点击整个卡片区域，跳转至 Reader 页面，并自动在 URL 附带该仓库的标识（如 `?repo=PRD-Reader`）。

**2.4 空状态/异常提示区 (Empty State Area - 仅在无数据时显示)**
*   **插画/图标**：表示“空”或“配置”的友好图形。
*   **提示文案**：例如“暂无可用仓库，请联系管理员在后台配置白名单”。

## 3. 阅读器页 (`index.html` / `Reader.tsx` 改动点)

*注：主阅读区、左侧抽屉式文件树等现有结构保持不变，本部分仅描述本次需求涉及的改动。*

**3.1 侧边栏顶部 / 仓库切换触发器区 (Sidebar Top Area)**
*   **触发器组件 (Dropdown Trigger)**：
    *   **位置**：集成在左侧抽屉式侧边栏 (Sidebar) 的最顶部，取代原有的静态仓库标识。
    *   **展示内容**：当前正在阅读的仓库名称（从 URL 的 `?repo=` 参数或 Store 中获取），前方可带仓库 Icon (☁️)。
    *   **视觉暗示**：名称右侧紧跟一个微小的向下箭头（Chevron Down）图标。
    *   **交互动作**：点击触发器，在下方弹出“下拉列表层”。

**3.2 下拉仓库列表层 (Dropdown Menu Overlay)**
*   **浮层容器**：一个带有轻微阴影、圆角（Notion 风格）的绝对定位浮层，悬浮在页面其他内容之上。
*   **列表内容区 (Menu Items)**：
    *   **渲染逻辑**：遍历系统配置的所有白名单仓库。
    *   **列表项 (Item)**：每行显示一个仓库名称 `name` 及子级路径 `path`。
    *   **选中状态反馈**：当前正在阅读的仓库项，其左侧或右侧必须展示一个对勾（Check）图标。
    *   **悬浮反馈**：鼠标经过列表项时，背景色微弱变化（如浅灰）。
    *   **交互动作**：点击非当前仓库项，触发仓库切换逻辑（更新 URL、清空文件参数、关闭下拉层、重新拉取文件树显示 Loading），并默认尝试加载根目录下的 `README.md`。
*   **分隔线 (Divider)**：在仓库列表下方，有一条细微的水平分割线。
*   **底部操作区 (Bottom Action Area)**：
    *   **返回按钮/链接**：位于分割线下方。文字如“← 返回所有项目”或“返回仓库列表”。
    *   **交互动作**：点击后，关闭下拉层，清空 URL 中的所有查询参数（跳转回 `home.html` 独立仓库列表页）。
