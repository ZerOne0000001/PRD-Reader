# 用户旅程地图：仓库切换与独立列表页 (F2_仓库切换)

| 阶段 (Stage) | 系统行为 (System Action) | 触点 (Touchpoints) | 机会点/痛点 (Opportunities/Pain Points) |
| :--- | :--- | :--- | :--- |
| **阶段 1：打开系统首页** | 拦截根路由，判断 URL 中是否携带特定文件参数。如果没有，则导向“独立仓库列表页”。 | 浏览器 URL 输入框、系统首页加载状态。 | **机会点**：对于初次访问的用户，可以直接展示清晰的系统说明或帮助文档；对于老用户，可以展示他们经常访问的仓库（如可能）。<br>**痛点**：若后台白名单未配置，需清晰提示并引导去后台配置。 |
| **阶段 2：浏览仓库列表** | 渲染白名单中的所有仓库（以卡片/列表形式展示），并根据后台配置加载仓库信息（如名称、描述）。 | 仓库列表页、各个仓库卡片项。 | **机会点**：可以增加搜索/过滤框，当仓库极多时，让 PM 能输入关键字快速定位；卡片上可以展示该仓库最后更新的时间或 README 摘要。 |
| **阶段 3：进入阅读器** | 用户点击特定仓库卡片后，系统获取该仓库的文件树并渲染 Reader 页面，URL 更新为带 `?repo=xxx` 参数，左侧树仅展示选中仓库的内容。 | 阅读器加载骨架屏、左侧文件树、右侧默认内容页。 | **机会点**：可以在右侧默认渲染仓库根目录下的 README.md，提升初次进入的体验。<br>**痛点**：若获取文件树超时或网络异常，需要有明确的重试或错误状态提示。 |
| **阶段 4：阅读中切换仓库** | 用户点击左上角的当前仓库名称，系统弹出一个下拉列表。用户在列表中点击其他仓库，触发新的仓库树获取与渲染。 | 阅读器左上角当前仓库名称（下拉组件）、下拉列表中的仓库项。 | **机会点**：下拉列表中可以高亮显示当前选中的仓库，并在列表项旁边加上仓库图标，增强辨识度。<br>**痛点**：如果仓库列表很长，下拉框可能会超出屏幕，需处理好滚动。 |
| **阶段 5：返回仓库列表页** | 用户在阅读器左上角的下拉列表中点击“返回选仓库页面”，系统清除 URL 的文档参数，跳转回首页。 | 阅读器左上角下拉列表中的“返回仓库列表”选项。 | **机会点**：通过明显的视觉隔离（如一条分割线或特殊图标）将“返回”操作与“切换仓库”操作区分开，防止误触。 |
