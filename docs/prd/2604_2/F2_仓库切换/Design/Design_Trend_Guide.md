# 视觉风格定义与设计趋势指南 (Design_Trend_Guide) - F2_仓库切换

## 1. 审美方向预判 (Aesthetic Direction)
基于蓝图规范（Blueprint），PRD-Reader 项目已确立了**极简清新风 (Notion 风格)**，并强调 **Soft & Pastel (柔和马卡龙色系)**。本期原型必须严格遵循这一既定风格，不能随意引入其他不相关的工业感或冰冷极简主义。

*   **Purpose (目的)**：打破传统工具的冰冷感，营造类似“个性化日记本”的舒适阅读体验。
*   **Tone (基调)**：轻松、圆润、可爱、柔和。
*   **Constraints (约束)**：全局隐藏滚动条（保留滚动功能），组件广泛使用大圆角（`12px` 或 `16px`），阴影必须是带有色彩倾向的柔和投影。

## 2. 核心视觉元素规范 (依据 Blueprint)

### 2.1 Typography (排版与字体)
*   **标题字体**：使用 `Quicksand` 字体，保持圆润感。
*   **正文字体**：使用 `Nunito` 字体，行高设定为 `1.7`。
*   **特色排版**：H1 标题底部带有半透明黄色高光下划线装饰，H2 标题前方带有粉色小花 `✿` 装饰符。

### 2.2 Color & Theme (色彩体系)
*   **背景色**：`--bg-pastel: #FDF4F5` (极浅粉色)。
*   **侧边栏背景**：`--sidebar-bg: #E8F3F1` (浅青色)。
*   **文字颜色**：`--text-dark: #4A4E69`。
*   **强调色**：使用 `--accent-blue` 和 `--accent-pink`。

### 2.3 Motion & Interaction (动态与交互)
*   **动效曲线**：所有的 Hover 和弹窗（包括下拉菜单）必须采用**弹性 (Spring) 曲线**，避免生硬的线性过渡。
*   **阴影 (Shadows)**：避免生硬的黑色投影，改用带有色彩倾向的投影（如 `rgba(162, 210, 255, 0.4)`）。
*   **特殊反馈**：选中状态需高亮蓝色背景。

### 2.4 Spatial Composition (空间构成)
*   **卡片与组件**：使用 `12px` 或 `16px` 的大圆角（Large Border Radius）。
*   **下拉菜单 (Dropdown)**：菜单本身应具备大圆角和柔和的色彩投影，悬浮交互需要带有弹簧效果。
