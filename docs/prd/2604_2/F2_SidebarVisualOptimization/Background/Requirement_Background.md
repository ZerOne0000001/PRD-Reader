# 需求背景与目标 (Requirement Background)

## 1. 业务目标与背景
* **业务背景**：在当前版本的 PRD Reader 侧边栏中，由于默认宽度限制和未做特殊处理，当文件夹或需求文件名称过长时会出现显示不全的问题。这严重影响了用户对长标题文档的辨识度，降低了阅读体验。
* **业务目标**：通过综合的视觉与交互优化（如增加基础宽度、缩小字号、截断与 Tooltip 提示、换行支持以及拖拽调整宽度等），彻底解决长文本显示不全的问题，提升侧边栏的可用性和阅读体验。

## 2. 用户画像 (User Personas)

| 角色 | 特征描述 | 核心痛点 | 行为限制/偏好 |
| :--- | :--- | :--- | :--- |
| **文档阅读者** (Reader) | 团队成员，需要频繁查阅 PRD 文档。 | 面对长文件名的需求文档，无法一眼看全标题，需要点击进去才能确认是不是自己要找的文件，效率低下。 | 偏好清晰直观的目录结构；希望能够根据自己的屏幕尺寸自由调整侧边栏宽度。 |
| **文档管理员** (Admin) | 负责整理和规范 PRD 目录。 | 在编写和整理长标题的 PRD 时，发现系统侧边栏展示效果差，可能因此被迫缩减标题长度，导致标题不够精确。 | 偏好系统能够自适应各种长度的规范命名，不需要为了显示效果妥协命名规范。 |

## 3. 核心场景 (Core Scenarios)

| 场景名称 | 关联用户 | 触发条件与环境 | 当前痛点 | 核心动作 (期望) | 期望结果 | 潜在异常 |
| :--- | :--- | :--- | :--- | :--- | :--- | :--- |
| **浏览长标题文件** | Reader/Admin | 用户在侧边栏浏览包含长标题的文件夹或文件。 | 标题被粗暴截断，无法识别完整含义。 | 1. 鼠标悬停查看完整标题。 2. 标题在侧边栏自动换行显示（如有必要）。 | 用户能够通过截断+Tooltip或换行的方式，清晰获取完整文件名。 | Tooltip 遮挡了相邻文件的点击区域。 |
| **自定义侧边栏宽度** | Reader/Admin | 用户觉得当前侧边栏宽度不够，想要展示更多文字内容。 | 侧边栏宽度固定，无法根据屏幕大小或个人偏好调整。 | 鼠标拖拽侧边栏边缘，自由调整其宽度。 | 侧边栏平滑拉伸，内容自适应展示更多字符。 | 拖拽过宽挤压主阅读区，或拖拽过窄导致内容完全不可见（需设置最大/最小宽度限制）。 |
