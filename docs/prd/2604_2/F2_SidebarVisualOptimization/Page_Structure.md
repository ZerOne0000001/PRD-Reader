# 信息架构与页面结构 (Page Structure)

## 全局导航与结构
本次需求主要对现有 `Reader` 页面内的侧边栏组件进行局部视觉与交互升级，不涉及新增页面。
*   **端侧定义**: 桌面端 (Desktop Only)
*   **影响页面**: `Reader` 页面 (预设 HTML 文件名：`reader_optimized.html` 用于原型展示)

## 页面分区拆解 (Reader 页面)

### 1. 侧边栏容器区 (Sidebar Container)
*   **默认状态**: 页面加载时，侧边栏占据左侧（或特定位置），宽度设置为新的默认值（如 280px）。
*   **拖拽调整手柄 (Resizer Handle)**: 
    *   **位置**: 位于侧边栏与主阅读区交界的边缘。
    *   **视觉与交互**: 默认状态下可能是一条细细的分割线或不可见（仅增加热区）。当鼠标悬停在其上（前后 2-3px 范围内）时，光标变为 `col-resize`（左右箭头）样式，并可能伴随轻微的视觉反馈（如线条变色加粗）。
    *   **拖拽动作**: 按住左键并左右移动鼠标，实时改变侧边栏宽度。松开左键完成调整，宽度被限制在最小 (200px) 和最大 (500px) 之间。

### 2. 目录树列表区 (Tree List Area)
*   **排版优化**: 整体字号调整为 13px，行高和图标间距调整以显得更加精致紧凑。
*   **树节点项 (Tree Node Item)**: 包含展开/折叠图标、文件类型图标和文件名称文本。
    *   **文本截断**: 节点文本容器采用 Flex 布局，文本元素设置 `flex: 1`, `min-width: 0`, `white-space: nowrap`, `overflow: hidden`, `text-overflow: ellipsis` 以实现尾部单行截断。
    *   **智能 Tooltip 交互**:
        *   当节点文本实际宽度大于可用宽度（发生截断）时，鼠标悬停在整个节点项上停留（如 300ms 延迟）后，在节点上方或下方弹出一个轻量级的 Tooltip 提示框，显示完整的长文本名称。
        *   Tooltip 样式应与系统暗黑/明亮主题保持一致（如暗色背景、白色文字，带有轻微的阴影和圆角）。
        *   当鼠标移出节点区域，或开始拖拽侧边栏宽度导致文本不再截断时，Tooltip 立即消失。

### 3. 主内容阅读区 (Main Content Area)
*   **自适应**: 该区域的宽度不固定，它会自动占据侧边栏剩余的屏幕空间 (`flex: 1` 或类似机制)。当侧边栏宽度被拖拽改变时，主区域的内容会自动重新排版适应新的宽度。
