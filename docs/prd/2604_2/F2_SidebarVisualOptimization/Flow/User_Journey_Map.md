# 用户旅程地图 (User Journey Map)

| 阶段 (Stage) | 触点 (Touchpoint) | 用户行为 (User Actions) | 痛点/挑战 (Pain Points/Challenges) | 系统响应 (System Response) | 机会点/优化策略 (Opportunities) |
| :--- | :--- | :--- | :--- | :--- | :--- |
| **1. 初始浏览** | 侧边栏目录树 | 打开文档页面，浏览侧边栏目录。 | 原有布局拥挤，大字号长文本容易显得杂乱。 | 呈现默认增宽后的侧边栏，采用更小的字号（如 13px）和优化的间距。 | 提供更清爽的基础视觉体验，增加首屏可展示的信息量。 |
| **2. 发现长文件名** | 长标题文件节点 | 视线停留在一个名称很长的文件或文件夹上。 | 文本被截断，无法直接读完。 | 超出宽度的文本平滑地显示省略号（...），未撑破布局。若是支持换行的层级，则优雅折行并保持缩进。 | 确保截断样式统一；换行方案需保证行高和点击区域的一致性。 |
| **3. 查看完整名称** | 截断的文件节点 | 将鼠标悬停在带有省略号的文件节点上。 | 需要快速确认这是不是自己要找的文件。 | 鼠标悬停短暂延迟后，弹出包含完整文件名的 Tooltip 提示框。 | Tooltip 样式应轻量，避免遮挡重要信息；延迟时间应适中（如 200-300ms）。 |
| **4. 调整视图** | 侧边栏边缘分隔线 | 发现即使有 Tooltip 还是想一次性看全所有层级，于是拖拽侧边栏右侧边缘。 | 找不到拖拽的抓手，或者拖拽过程卡顿。 | 鼠标移动到分隔线时变为拖拽光标，按住并拖动时，侧边栏宽度实时跟随改变，直到达到最大/最小限制。 | 增加隐形的拖拽热区宽度（如前后各扩宽 2-3px 触发区）以提升可拖拽性；确保拖拽动画流畅。 |
| **5. 结束浏览** | 侧边栏与主区域 | 调整好合适的侧边栏宽度后，点击文件进入阅读。 | 刷新页面后侧边栏宽度重置，需要重新调整。 | （可选增强）系统记录用户的侧边栏宽度偏好，下次打开时保持该宽度。 | 提升个性化体验。 |
