# 设计趋势与视觉风格定义 (Design Trend Guide)

## 1. 核心视觉论点 (Visual Thesis)
**Soft & Pastel (亲和马卡龙与圆润质感)**
“延续系统现有的柔和色系（浅青色与极浅粉色交织），通过大圆角、低对比度的彩色投影和圆润的排版，打造一种极具亲和力、轻松愉悦的文档阅读体验，打破传统工具的冰冷感。”

## 2. 四个维度定义
*   **Purpose (设计目的)**: 在保持系统既有柔和风格的前提下，解决长文本截断问题。增加交互的趣味性和反馈的柔和度。
*   **Tone (基调)**: 轻松、圆润、可爱。类似于具有个性化装扮的日记本，而非冷冰冰的代码编辑器。
*   **Constraints (约束)**: 
    *   **色彩约束**: 严格遵循系统定义的 CSS 变量：背景使用 `--bg-pastel: #FDF4F5`，侧边栏使用 `--sidebar-bg: #E8F3F1`，文字使用 `--text-dark: #4A4E69`。强调色使用 `--accent-blue` 和 `--accent-pink`。
    *   **圆角与排版**: 广泛使用大圆角（如节点 `12px` 或 `16px`，Tooltip `16px`）。字体统一使用 `Nunito`（正文）和 `Quicksand`（标题），以保持圆润感。
    *   **阴影 (Shadows)**: 避免使用生硬的黑色投影，改用带有色彩倾向的柔和投影（如 `rgba(162, 210, 255, 0.4)`）。
*   **Differentiation (差异化体验)**:
    *   **Pill-shape Tooltip (胶囊提示)**: Tooltip 放弃传统的方正黑底，采用白底、大圆角、带有轻柔蓝色投影的胶囊形状，文字颜色保持 `--text-dark`，与整体的 Pastel 风格完美融合。
    *   **Soft Resizer (柔和拖拽条)**: 拖拽条在 Hover 时不再是生硬的实线，而是一条带有一定宽度和极低透明度的主题色光晕，给予用户“软糯”的交互反馈。

## 3. 交互论点 (Interaction Thesis)
*   **弹性动效**: 所有的 Hover、Active 和 Tooltip 出现动画，都应采用类似于弹性 (Spring) 的 `cubic-bezier` 曲线，而非线性的生硬过渡。
*   **智能防抖**: Tooltip 依然保持按需显示（仅在截断时）和 300ms 的防抖延迟，确保阅读时不被打扰，但出现的瞬间是轻快可爱的。
