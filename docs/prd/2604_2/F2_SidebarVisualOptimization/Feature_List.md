# 功能清单 (Feature List)

| 一级模块 | 二级功能 | 功能描述 | 优先级 (P0-P3) |
| :--- | :--- | :--- | :--- |
| **基础视觉优化** | 侧边栏默认宽度调整 | 将 Reader 页面侧边栏的默认基础宽度从原有值（如 250px）增加到一个更合适的值（如 280px 或 300px），以容纳更多文本。 | P0 |
| **基础视觉优化** | 目录树排版调整 | 适当缩小侧边栏目录树文本的字号（如从 14px 调整为 13px），优化行高和图标间距，使排版更紧凑精致。 | P1 |
| **长文本处理** | CSS 单行截断 | 对所有目录节点应用纯 CSS 的单行截断（`text-overflow: ellipsis`），确保超长文本不会撑破侧边栏布局，统一在尾部显示省略号。 | P0 |
| **长文本处理** | 智能按需 Tooltip | 引入自定义的 Tooltip 组件。仅当节点文本实际发生 CSS 截断时，鼠标悬停（且停留如 300ms 延迟防抖后）才显示包含完整文件名的 Tooltip 提示框。 | P0 |
| **长文本处理** | Tooltip 交互与定位 | Tooltip 的出现位置应智能计算（如跟随鼠标或固定在元素上方），避免被遮挡。鼠标移开元素区域后 Tooltip 立即消失。 | P0 |
| **交互增强** | 侧边栏拖拽调宽 | 在侧边栏右边缘增加一条可拖拽的分隔线（Resizer）。用户鼠标悬停时显示水平调整光标，按住可实时左右拖拽调整侧边栏宽度。 | P1 |
| **交互增强** | 拖拽边界限制 | 为拖拽功能设置最小宽度限制（如 200px）和最大宽度限制（如 500px 或主区域的 50%），防止布局崩坏。 | P1 |
| **交互增强** | (可选) 宽度偏好记忆 | 将用户拖拽调整后的自定义侧边栏宽度持久化存储（如 LocalStorage），刷新页面或下次进入系统时恢复该宽度。 | P2 |
